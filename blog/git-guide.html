Git Guide

THS ASSUMES THAT GIT IS INSTALLED AND CONFIGURED

$ git config --global user.name 'Your Name' $ git config --global user.email you@somedomain.com

GIT - Local Repository

Git is a version tracking system to track the development of a project. Think of your project as a folder having a tree-like structure with sub-folders and files. You want to track changes you make across the tree. Git creates 3 trees for this purpose:

  1. Working folder (which has your files);
  2. Index tree (a staging area for proposed changes);
  3. Head tree (to use when committing these changes).

Git workflow: Essentially, you are taking a snapshot of your folder-tree at a given time and saving it locally. You can save this snapshot remotely as a backup (see below for Github)

Getting a Git repository To do anything in Git you have to have a Git repository. This is where Git stores the data for the snapshots you are saving. There are 2 ways to get a Git repository:

  1. initialize a new one from an existing directory (e.g., new projects); OR
  2. clone an existing public repository (e.g., wanted to copy or work on someone else's project)

Initializing a new repo Let's say you have a new project that you want to track in a folder called "HelloWorld". First, cd into that folder (Need to do only ONCE for each project), and then initialize git $ cd HelloWorld $ git init .

Other useful commands

 $ git init

 $ git init <directory>

Cloning a Git repo Let's say you want to copy a project or collaborate with someone on a project or use their code, you can clone it ($ git clone [url]). For example, if you want to clone the MetaMoo web app from Github

$ git clone git@github.com:MetaMoo/MetaMooWebApp.git

If you are cloning a local repo (e.g., maybe creating a working copy)

$ git clone /path/to/repository

Adding and committing to the repo

  1. Propose changes to the index tree - take all untracked files and start tracking them

    $ git add --all 2. Commit changes to the head tree - Take a snapshot of these files and save it as a checkpoint. Choose a good message to describe the commit and the work you have done for it to help with recall later.

    $ git commit -am "Commit message"

Seeing what has been modified and committed To see the state of the project, we can use:

$ git status

Github - Remote Repository

Github is a service providing a remote repository for Git. Essentially, you can version control and share your projects with everyone. Also, it is useful if you are changing computers or simply want a backup.

Now, you can (1) either create a new remote repository and add your local repo to it; or (2) add to an existing repo (e.g., the one you cloned from)

To create a new remote repo:

  1. Sign in to Github and create a new repo (choose SSH option so you don't have to keep entering your password each time you send something)
  2. Hook up your local Git repo to the remote Github repo $ git remote add origin git@github.com:appname/appname.git
  3. Push for the first time to Github. *Note: The -u to git push means "as well as pushing, associate my master branch with the one I'm pushing to". $ git push -u origin master
  4. Subsequent pushes you don't need the "-u" option

To add files to an existing repo that you cloned from, simply push: $ git push origin master

Note: "master" points to the main branch. You can change "master" to whatever branch you want.
